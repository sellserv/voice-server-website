Self-Hosting Guide

Deploy your own instance with Docker in minutes.

Docker (Recommended)

The fastest way to get running. Requires Docker installed on your server.

# Download the compose file
curl -O https://raw.githubusercontent.com/sellserv/voice-server/main/docker-compose.yml

# Edit the environment variables
nano docker-compose.yml
# Set JWT_SECRET (run: openssl rand -hex 32)
# Set MEDIASOUP_ANNOUNCED_IP to your server's public IP

# Start the server
docker compose up -d

Your instance will be available at http://your-server-ip:3000

Updating

docker compose pull
docker compose up -d

Data and uploads persist across updates via Docker volumes.

Admin Setup

  1. Register an account on your instance
  2. Verify your email
  3. Add your username to ADMIN_USERS in the compose file
  4. Restart: docker compose up -d

Good to Know

  • SQLite — No separate database server needed. Data is a single file. Works great for small-medium instances.
  • WebRTC (mediasoup) — Requires UDP ports 40000-40100 open. Voice won't work behind restrictive firewalls that block UDP.
  • Email — Strongly recommended. Without it, users can't receive MFA codes or reset passwords. Resend has a generous free tier.
  • Reverse Proxy — For production, put nginx or Caddy in front for SSL termination. The Docker image only serves HTTP on port 3000.
  • All features are free — Self-hosted instances have full access to every feature. The Pro tier only applies to the official instance.
  • Updates — Database migrations run automatically on startup. Just pull the new image and restart.
  • Backups — Back up the data/ and uploads/ volumes. That's your entire instance.
