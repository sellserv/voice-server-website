Prerequisites

  • Node.js 22 or later (nodejs.org)
  • npm — comes with Node.js
  • Ubuntu 24.04 LTS or Debian 12+ recommended for production

Quick Start (Development)

Clone the repository and get started locally:

git clone https://github.com/sellserv/voice-server.git
cd voice-server

# Install dependencies
npm install

# Copy and configure environment
cp .env.example .env

# Generate a secure JWT secret
sed -i "s/change-me-to-a-random-string/$(openssl rand -hex 32)/" .env

# Run dev mode (server + client with hot reload)
npm run dev

Open http://localhost:5173 to access the app. See Admin Setup to configure your admin account.

Setup Script (Production)

The setup script (deploy/setup.sh) configures a fresh server with everything you need:

  • System updates + 1GB swap + automatic security updates
  • Node.js 22 via NodeSource
  • Build tools for native module compilation
  • nginx as a reverse proxy with Cloudflare IP allowlist, security headers, TLS hardening, and rate limiting
  • UFW firewall (SSH, HTTP, HTTPS, WebRTC UDP ports)
  • SSH hardening (password auth disabled, root login restricted to key-only)
  • fail2ban for brute-force protection
  • Dedicated voip-server system user
  • .env with auto-generated JWT secret
  • SSL certificate directory with restricted permissions
  • systemd service with full sandboxing (syscall filters, capability restrictions, resource limits)

Running the Script

# Clone to your server
git clone https://github.com/sellserv/voice-server.git /opt/voip-server
cd /opt/voip-server

# IMPORTANT: Edit DOMAIN at the top of the script first
nano deploy/setup.sh

# Run the setup script
sudo bash deploy/setup.sh

# Edit your config
nano .env
# Set MEDIASOUP_ANNOUNCED_IP to your server's public IP
# Set CORS_ORIGINS to https://your-domain.com

# Install, build, and start
npm ci --omit=dev
npm run build
sudo systemctl start voip-server

Environment Configuration

After running the setup script, your .env file is at /opt/voip-server/.env. The script auto-generates a JWT secret. You still need to configure:

Required

# Set to your server's public IP — required for WebRTC voice
MEDIASOUP_ANNOUNCED_IP=203.0.113.50

# Set to your domain
CORS_ORIGINS=https://your-domain.com

Recommended

# Email (required for login — MFA codes are sent via email)
RESEND_API_KEY=re_xxxxxxxxxxxx
EMAIL_FROM=noreply@your-domain.com

# GIF picker
GIPHY_API_KEY=your-giphy-api-key

After editing, restart the service: sudo systemctl restart voip-server

Admin Setup

Instance admins are configured via the ADMIN_USERS environment variable — a comma-separated list of usernames with full platform control.

  1. Register an account at your server URL
  2. Verify your email address
  3. Add your username to ADMIN_USERS in .env: 
    ADMIN_USERS=yourusername
  4. Restart the server: 
    sudo systemctl restart voip-server
  5. Your account now has instance admin access — create invite codes from the admin panel to let others join

Multiple admins can be set with a comma-separated list: ADMIN_USERS=admin1,admin2

Email Setup (Resend)

Email is required in production for MFA login codes, email verification, password resets, and account lockout notifications. Without it, the server will refuse to start in production mode.

  1. Create an account at resend.com
  2. Add and verify your domain under Domains (requires adding DNS records)
  3. Create an API key under API Keys
  4. Add to your .env: 
    RESEND_API_KEY=re_xxxxxxxxxxxx
EMAIL_FROM=noreply@your-domain.com

The EMAIL_FROM address must use the domain you verified in Resend.

Cloudflare Setup

  1. Point your domain to your server's IP with Proxy ON (orange cloud)
  2. Go to SSL/TLS > Origin Server and create an Origin Certificate for your domain
  3. Save the certificate to /etc/ssl/voip-server/origin.pem
  4. Save the private key to /etc/ssl/voip-server/origin-key.pem
  5. Set SSL/TLS mode to Full (strict)

Cloudflare Turnstile (CAPTCHA)

Turnstile adds CAPTCHA protection to the registration form, preventing automated bot signups. Optional — if not configured, registration works without CAPTCHA.

  1. Go to the Cloudflare Turnstile dashboard
  2. Click Add site and enter your domain
  3. Choose widget mode (Managed recommended)
  4. Copy the Site Key and Secret Key
  5. Add both to your .env: 
    TURNSTILE_SITE_KEY=0x4AAAAAAA...
TURNSTILE_SECRET_KEY=0x4AAAAAAA...
  6. Restart the server: sudo systemctl restart voip-server

Once configured, the registration page will show a Turnstile challenge automatically.

Auto-Deploy (GitHub Actions)

Pushes to main automatically deploy to your server via GitHub Actions. The workflow pulls latest code, installs dependencies, builds the client, restarts the service, and runs a health check.

Setup

  1. Create a deploy user on your server: 
    sudo useradd -m -s /bin/bash deploy
sudo usermod -aG voip-server deploy
echo 'deploy ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart voip-server' \
  | sudo tee /etc/sudoers.d/voip-server-deploy
  2. Generate an SSH key pair and copy the public key to your server: 
    ssh-keygen -t ed25519 -f ~/.ssh/voip-server-deploy -N ""
ssh-copy-id -i ~/.ssh/voip-server-deploy.pub deploy@your-server-ip
  3. Add these repository secrets in GitHub (Settings > Secrets > Actions):
    SecretValue
    DEPLOY_HOSTYour server's IP address
    DEPLOY_USERdeploy
    DEPLOY_SSH_KEYContents of the private key file
  4. Ensure the repo is cloned at /opt/voip-server and the deploy user has read access.

GIPHY API

The GIF picker and GIF banner selection require a GIPHY API key.

  1. Go to developers.giphy.com and create an app
  2. Copy the API key
  3. Add to your .env: 
    GIPHY_API_KEY=your-api-key-here
  4. Restart the service: sudo systemctl restart voip-server
