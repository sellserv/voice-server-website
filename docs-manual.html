Manual Install

Install directly on Ubuntu 24.04 LTS or Debian 12+ with full control.

Prerequisites

  • Ubuntu 24.04 LTS or Debian 12+ (x86_64)
  • Root or sudo access
  • A domain name pointed to your server

What the Setup Script Does

The included deploy/setup.sh configures a fresh server with:

  • System updates + 1GB swap + automatic security updates
  • Node.js 22 via NodeSource
  • Build tools for native module compilation (mediasoup)
  • nginx reverse proxy with security headers, TLS hardening, and rate limiting
  • UFW firewall (SSH, HTTP, HTTPS, WebRTC UDP ports)
  • SSH hardening (password auth disabled, key-only root login)
  • fail2ban for brute-force protection
  • Dedicated voip-server system user
  • .env with auto-generated JWT secret
  • systemd service with full sandboxing

Installation

1. Clone the Repository

git clone https://github.com/sellserv/voice-server.git /opt/voip-server
cd /opt/voip-server

2. Edit and Run the Setup Script

# Set your domain at the top of the script
nano deploy/setup.sh

# Run the script
sudo bash deploy/setup.sh

3. Configure Environment

nano .env

Required:

# Your server's public IP — required for WebRTC voice
MEDIASOUP_ANNOUNCED_IP=203.0.113.50

# Your domain
CORS_ORIGINS=https://your-domain.com

Recommended:

# Email (required for MFA login codes)
RESEND_API_KEY=re_xxxxxxxxxxxx
EMAIL_FROM=noreply@your-domain.com

# GIF picker
GIPHY_API_KEY=your-giphy-api-key

4. Install, Build, and Start

npm ci --omit=dev
npm run build
sudo systemctl start voip-server

Admin Setup

  1. Register an account at your server URL
  2. Verify your email
  3. Add your username to ADMIN_USERS in .env: 
    ADMIN_USERS=yourusername
  4. Restart: sudo systemctl restart voip-server

SSL Setup (Cloudflare)

  1. Point your domain to your server's IP with Proxy ON (orange cloud)
  2. Go to SSL/TLS > Origin Server and create an Origin Certificate
  3. Save the certificate to /etc/ssl/voip-server/origin.pem
  4. Save the private key to /etc/ssl/voip-server/origin-key.pem
  5. Set SSL/TLS mode to Full (strict)

Email Setup (Resend)

Email is required for MFA login codes, email verification, password resets, and account lockout notifications.

  1. Create an account at resend.com
  2. Add and verify your domain under Domains
  3. Create an API key under API Keys
  4. Add to .env and restart

CAPTCHA (Optional)

Add Cloudflare Turnstile to prevent automated signups:

  1. Go to the Turnstile dashboard and add your site
  2. Copy the Site Key and Secret Key
  3. Add both to .env: 
    TURNSTILE_SITE_KEY=0x4AAAAAAA...
TURNSTILE_SECRET_KEY=0x4AAAAAAA...
  4. Restart the service

Auto-Deploy (GitHub Actions)

Automatically deploy on push to main:

  1. Create a deploy user: 
    sudo useradd -m -s /bin/bash deploy
sudo usermod -aG voip-server deploy
echo 'deploy ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart voip-server' \
  | sudo tee /etc/sudoers.d/voip-server-deploy
  2. Generate and copy an SSH key: 
    ssh-keygen -t ed25519 -f ~/.ssh/voip-deploy -N ""
ssh-copy-id -i ~/.ssh/voip-deploy.pub deploy@your-server-ip
  3. Add repository secrets in GitHub:
    SecretValue
    DEPLOY_HOSTServer IP address
    DEPLOY_USERdeploy
    DEPLOY_SSH_KEYPrivate key contents

Updating

cd /opt/voip-server
git pull origin main
npm ci --omit=dev
npm run build
sudo systemctl restart voip-server

Backups

Back up these two directories to save your entire instance:

  • /opt/voip-server/data/ — SQLite database
  • /opt/voip-server/uploads/ — User-uploaded files
