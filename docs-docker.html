Docker Setup

Deploy SellServ Voice with Docker Compose in minutes.

Prerequisites

  • Docker installed (includes Compose)
  • A server with a public IP address
  • UDP ports 40000-40100 open in your firewall

1. Download the Compose File

curl -O https://raw.githubusercontent.com/sellserv/voice-server/main/docker-compose.yml

2. Configure Environment

Open the file and set the required variables:

nano docker-compose.yml

Required:

  • JWT_SECRET — Generate with openssl rand -hex 32
  • MEDIASOUP_ANNOUNCED_IP — Your server's public IP address

Recommended:

  • CORS_ORIGINS — Your domain (e.g. https://chat.example.com)
  • ADMIN_USERS — Your username for instance admin access
  • RESEND_API_KEY + EMAIL_FROM — For email MFA and verification

3. Start the Server

docker compose up -d

Your instance is now running at http://your-server-ip:3000

4. Admin Setup

  1. Register an account on your instance
  2. Verify your email (if email is configured)
  3. Add your username to ADMIN_USERS in the compose file
  4. Restart: docker compose up -d

Updating

docker compose pull
docker compose up -d

Data and uploads persist across updates via Docker volumes. Database migrations run automatically on startup.

Reverse Proxy (Production)

For production, put nginx or Caddy in front for SSL termination. The Docker container serves HTTP on port 3000. Example nginx config:

server {
    listen 443 ssl http2;
    server_name chat.example.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

Backups

Back up the Docker volumes to save your entire instance:

# Find volume paths
docker volume inspect sellserv-data sellserv-uploads

# Or copy from the container
docker cp sellserv:/app/data ./backup-data
docker cp sellserv:/app/uploads ./backup-uploads
